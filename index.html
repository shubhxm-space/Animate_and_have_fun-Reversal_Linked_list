What is this about?

What is a Linked List?

Unlike arrays that sit together in memory, a linked list is a chain of nodes spread out. Each node holds some data and an arrow (next pointer) pointing to the following node.

What does Reversing mean?

Reversing simply means flipping all the arrows so they point backward. The former tail becomes the new head!

Why do it?

It's a foundational mental model. If you can reverse a linked list, you understand pointers, references, and holding temporary state.

The 4 Magic Steps

  1. Save next: next = curr.next (Don't lose the rest of the list!)
  2. Reverse link: curr.next = prev (Flip the arrow backward)
  3. Move prev: prev = curr (Step forward)
  4. Move curr: curr = next (Step forward)
Time Complexity O(N) We visit each node exactly once
Space Complexity O(1) We only use 3 pointers, no matter the list size
Common Mistake: Modifying curr.next before saving it! If you change the link first, you sever the chain and lose the remaining list forever.

Pro Tips

💡

Draw it on paper

Always draw boxes and arrows before coding. It prevents off-by-one errors.

🔗

Reverse one connection at a time

Focus purely on isolating the current node and flipping its arrow to point backward.

🧪

Test edge cases

What if the list is empty (head = null)? What if it has one node? Test these mentally.

Knowledge Check

1. What is the final head of the list after reversal?
2. What happens if the list has only one node?
3. What is the correct initial state before the loop?
